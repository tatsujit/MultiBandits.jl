推定器 (Estimators)

型

MultiBandits.AbstractEstimator — Type
AbstractEstimator

全推定器の基底抽象型。サブタイプは以下のフィールドを持つ必要がある:

  • n_arms::Int — 腕の数
  • W::Vector{Float64} — 行動選択に使用する行動価値ベクトル
source
MultiBandits.EmptyEstimator — Type
EmptyEstimator <: AbstractEstimator

何も推定しないプレースホルダー推定器。RandomResponding 方策と組み合わせて使用する。

source
MultiBandits.EmpiricalReward — Type
EmpiricalReward <: AbstractEstimator

標本平均による行動価値推定器。各腕の報酬平均を Q 値として維持する。

キーワード引数

  • Q0::Float64=0.0 — Q 値の初期値

例

est = EmpiricalReward(4)       # 4本腕、Q0=0.0
est = EmpiricalReward(4; Q0=Inf)  # 楽観的初期値
source
MultiBandits.ThompsonSampling — Type
ThompsonSampling <: AbstractEstimator

Beta 分布によるトンプソンサンプリング推定器。Greedy 方策と組み合わせて使用する。

成功/失敗カウント（α, β）を更新し、Beta(α, β) からサンプリングした値を行動価値 W とする。

例

est = ThompsonSampling(4)  # 4本腕、Beta(1,1) で初期化
source
MultiBandits.ThompsonSamplingN — Type
ThompsonSamplingN <: AbstractEstimator

Beta 分布から samplesize 回サンプリングし、その平均を行動価値とするトンプソンサンプリング。 samplesize が大きいほど探索が減り、搾取寄りになる。

キーワード引数

  • samplesize::Int=1 — Beta 分布からのサンプル数（1 なら通常の ThompsonSampling と同等）
source
MultiBandits.ThompsonSamplingNU — Type
ThompsonSamplingNU <: AbstractEstimator

Beta 分布から複数回サンプリングし、非線形効用関数を適用するトンプソンサンプリング。 Q は標本平均、W = uf(Q) となる。

キーワード引数

  • samplesize::Int=1 — Beta 分布からのサンプル数
  • uf::Function=identity — 効用関数 (Q → W の変換)
source
MultiBandits.UCB1 — Type
UCB1 <: AbstractEstimator

UCB1 アルゴリズムによる推定器。W = Q + c√(ln t / N) をGreedy方策で使用する。

キーワード引数

  • Q0::Float64=Inf — 未選択腕を優先探索するための初期値
  • c::Float64=√2 — 探索ボーナスの係数

例

est = UCB1(4)          # 4本腕、デフォルト設定
est = UCB1(4; c=1.0)   # 探索係数を変更
source
MultiBandits.STEP — Type
STEP <: AbstractEstimator

閾値超過確率による満足化 (Satisficing with Threshold Exceeding Probability) 推定器。 Beta 分布の右裾確率 P(X ≥ aleph) を行動価値 W とする。Greedy 方策と組み合わせて使用する。

引数

  • n_arms::Int — 腕の数
  • aleph::Float64 — 要求水準（閾値）

例

est = STEP(4, 0.5)  # 4本腕、閾値 0.5
source
MultiBandits.RS — Type
RS <: AbstractEstimator

リスク感応型満足化 (Risk-Sensitive Satisficing) 推定器。 行動価値は W[a] = N[a] * (Q[a] - aleph) で計算される。Greedy 方策と組み合わせて使用する。

引数

  • n_arms::Int — 腕の数
  • aleph::Float64 — 要求水準（aspiration level）

例

est = RS(4, 0.5)  # 4本腕、要求水準 0.5
source
MultiBandits.CognitiveEstimator — Type
CognitiveEstimator <: AbstractEstimator

認知モデルに基づく多パラメータ推定器。学習率、忘却、固執性、効用関数（Beta CDF）を統合する。

総合行動価値は W = βV + φC で計算される（V は効用変換後の Q 値、C は固執性）。

主要キーワード引数

  • α::Float64=-1.0 — 学習率（-1.0 で標本平均）
  • αm::Float64=-1.0 — RPE 負の場合の学習率（-1.0 で α と同じ）
  • β::Float64=1.0 — 行動価値の逆温度
  • αf::Float64=0.0 — 忘却率（0.0 で忘却なし）
  • μ::Float64=0.0 — 忘却デフォルト値
  • τ::Float64=0.0 — 固執性の更新率
  • φ::Float64=0.0 — 固執性の逆温度
  • η::Float64=0.5 — Beta CDF のパラメータ（期待値）
  • ν::Float64=2.0 — Beta CDF のパラメータ（精度）

例

est = CognitiveEstimator(4; α=0.1, β=3.0)            # 固定学習率 + 逆温度
est = CognitiveEstimator(4; α=0.3, αf=0.1, μ=0.5)    # 忘却あり
est = CognitiveEstimator(4; τ=0.5, φ=1.0)             # 固執性あり
est = CognitiveEstimator(4; η=0.3, ν=5.0, α=0.5)     # Beta CDF 効用変換
source

MultiBandits.selection_probabilities — Function

W は K 行動それぞれの（総合的）行動価値ベクトル [W[1], W[2], ..., W[K]] == [1.2, 0.5, ..., 1.0, ] available_arms はその trial で選択可能だった腕（インデックス）のベクトル、例: [9, 2, 5, 1, 4] 総合的、というのは、 W = βV + φC であり、V は効用関数で変調された（かもしれない） Q 値、 C は固執性を表すベクトル

戻り値については、例えば確率ベクトル： [ 0.1, 0.2, 0.0, 0.1, 0.3, 0.0, 0.0, 0.0, 0.3] を返す。これは、 [9, 2, 5, 1, 4]に対応するブールベクトル [true, true,false, true, true,false,false,false, true] に対応している。

こうすることでデバッグも可能である。つまり、尤度が0.0になるようであれば、 available_arms 以外の腕が選ばれているということになる。

source
selection_probabilities(W::Vector{Float64}) -> Vector{Float64}

行動価値ベクトル W から softmax で選択確率を計算する（全腕対象）。

source
Compute selection probabilities for each action based on the estimator's action values.
Note that for GreedyPolicy, this will return a vector where the action(s) with the highest 
value have equal probability, and all others have zero probability.

NOTE: probably a small probability ϵ should be added to zero probability actions to allow likelihood calculation?
source

Policy >: Softmax の持っている β を無視する（実質 Softmax という型だけ参照している） W というのは行動効用 V と固執性 C のそれぞれに、それぞれの逆温度 β, φ をかけた合計のベクトル。 Estimator has its own β, so we use estimator.β instead of policy.β (= 1.0).
